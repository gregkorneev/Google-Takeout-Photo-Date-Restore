# Google Takeout Photo Date Restore

Скрипт для восстановления **файловой даты** фото и видео из JSON-метаданных, которые Google Photos добавляет в выгрузку через **Google Takeout**.

Проект предназначен для случаев, когда после экспорта из Google Photos рядом с каждым медиафайлом лежит JSON-файл, например:

```text
IMG_0001.jpg
IMG_0001.jpg.json
VID_0002.mp4
VID_0002.mp4.json
```

Скрипт читает timestamp из JSON и устанавливает его как дату доступа и изменения файла через `os.utime()`.

---

## Что делает скрипт

Текущая версия проекта:

- принимает путь к папке с файлами;
- ищет в этой папке медиафайлы и JSON-файлы;
- сопоставляет пары вида `filename.ext` ↔ `filename.ext.json`;
- берёт дату из:
  - `photoTakenTime.timestamp`, если поле есть;
  - иначе из `creationTime.timestamp`;
- меняет дату файла через `os.utime()`;
- выводит в консоль, какие файлы обработаны, пропущены или вызвали ошибку.

---

## Что **не** делает текущая версия

Важно понимать ограничения именно этой реализации:

- **не обрабатывает подпапки рекурсивно**;
- **не удаляет JSON** после обработки;
- **не пишет дату внутрь EXIF** фото;
- **не создаёт лог-файл**;
- работает только с JSON-файлами формата `имя_файла.расширение.json`;
- не поддерживает случай `IMG_0001.json`, если расширение медиа не входит в имя JSON.

## Структура проекта

```text
.
├── update_dates.py
└── process_photos.bat
```

### Файлы проекта

#### `update_dates.py`
Основной Python-скрипт.

#### `process_photos.bat`
Windows-обёртка для запуска скрипта двойным кликом.

Батник:
- переключает консоль в UTF-8;
- автоматически берёт папку, в которой лежит сам `.bat`;
- запускает `update_dates.py` для этой же папки.

---

## Как работает алгоритм

1. Пользователь передаёт путь к папке.
2. Скрипт получает список всех файлов в этой папке.
3. Отдельно собирает:
   - медиафайлы;
   - JSON-файлы.
4. Для каждого медиафайла проверяет, есть ли рядом JSON с тем же именем.
5. Если JSON найден:
   - открывает его;
   - извлекает timestamp;
   - меняет дату медиафайла.
6. Если JSON нет или дата в нём отсутствует — файл пропускается.
7. В конце выводится итог: сколько файлов обновлено и сколько пропущено.

## Поддерживаемые форматы

Скрипт обрабатывает следующие расширения:

- `.jpg`
- `.jpeg`
- `.png`
- `.heic`
- `.webp`
- `.mp4`
- `.mov`
- `.avi`
- `.mkv`
- `.3gp`

Список можно изменить прямо в коде в переменной `media_extensions`.

## Быстрый старт

### Вариант 1. Windows, запуск двойным кликом

1. Установи Python.
2. Положи файлы `update_dates.py` и `process_photos.bat` в папку с выгрузкой Google Takeout.
3. Убедись, что рядом лежат фото/видео и JSON-файлы.
4. Запусти `process_photos.bat` двойным кликом.

Скрипт обработает **только текущую папку**, где лежит батник.

---

### Вариант 2. Ручной запуск через терминал

#### Windows

```bat
python update_dates.py "C:\path\to\TakeoutFolder"
```

#### macOS / Linux

```bash
python3 update_dates.py "/path/to/TakeoutFolder"
```

## Как пользоваться правильно

### Рекомендуемый порядок работы

1. **Сделай резервную копию** папки с исходными файлами.
2. Помести скрипт в папку, которую хочешь обработать.

5. Только после этого запускай обработку на всей папке.

## Важные замечания

### 1. Меняется именно файловая дата

Скрипт изменяет:
- `mtime` — дату изменения файла;
- `atime` — дату доступа.

Он **не записывает дату внутрь EXIF** изображения.

Это означает, что:
- в файловом менеджере дата может отображаться правильно;
- но некоторые программы всё равно будут ориентироваться на внутренние метаданные фото, а не на файловую дату.

---

### 2. Обработка только одной папки

Скрипт использует `os.listdir()` и не проходит по подпапкам.

Если Google Takeout разбил файлы по нескольким папкам, текущую версию нужно запускать отдельно для каждой папки.

---

### 3. Жёстко прописанный путь к Python в `.bat`

В `process_photos.bat` используется конкретный путь:

```bat
"C:\Users\korne\AppData\Local\Programs\Python\Python313\python.exe"
```

Это удобно на авторском компьютере, но для публикации на GitHub лучше заменить на более универсальный вариант:

```bat
python update_dates.py "%folder%"
```

или

```bat
py update_dates.py "%folder%"
```

Иначе у других пользователей батник может не запуститься.

---

### 4. Формат JSON должен совпадать с именем файла

Текущая реализация ожидает именно такую схему:

```text
IMG_0001.jpg
IMG_0001.jpg.json
```

Если JSON называется так:

```text
IMG_0001.jpg
IMG_0001.json
```

то этот вариант текущая версия не обработает.

---


### Ошибка из-за пробелов в пути

Передавай путь в кавычках:

```bash
python3 update_dates.py "/Users/name/My Photos/Takeout"
```

---

### Файл пропущен, хотя JSON рядом есть

Проверь:

- совпадает ли имя файла и JSON;
- есть ли в JSON поле `photoTakenTime.timestamp` или `creationTime.timestamp`;
- не отличается ли расширение или регистр имени.

---

### Дата не изменилась в приложении просмотра фото

Скорее всего, приложение использует не файловую дату, а EXIF/внутренние метаданные. Для этого нужна отдельная версия инструмента с записью EXIF.

- восстановить порядок фото и видео после выгрузки из Google Photos;

- вернуть более адекватные даты файлам перед переносом в другую медиатеку;
- быстро обработать папку с Takeout без установки сторонних библиотек.
